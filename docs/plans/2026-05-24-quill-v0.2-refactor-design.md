# Quill v0.2 — Refactor Design

Date: 2026-05-24
Status: Approved (design phase)

## Goal

Improve the Quill plugin/skill across four dimensions that share root causes:

1. **Kill duplication** — workflow logic is currently written out three times (commands, Codex workflows, agent) and has already drifted.
2. **Tighten compression** — the compression contract is leaky: every write/revise reads and rewrites the *entire* `quill.json`, which holds all summaries and history.
3. **Correctness & tooling** — `actual_word_count` is an LLM guess, there is no schema validation/versioning, and the one real script has no tests.
4. **Polish & consistency** — "Quill" vs "Book Studio" naming, `allowed-tools` mismatch, inaccurate activation wording.

Decisions locked during brainstorming:
- **Full state restructure** (per-chapter summary files), not the conservative in-`quill.json` variant.
- **"Quill"** as the single product name everywhere.
- Summary files live in **hidden `.quill/summaries/`**.
- **Add GitHub Actions CI** to run the script tests.

---

## 1. Single source of truth (kills duplication)

`skills/quill-context/references/` becomes the only full copy of the logic, split so heavy material loads only when needed:

- `workflows.md` — the 9 workflows (the canonical spec). Stays lean.
- `schema.md` — `quill.json` schema + per-chapter summary schema + `schema_version` + validation rules.
- `init-templates.md` — README templates, `book.tex` preamble, directory layout. Loaded only during init.
- `format-conversion.md` — detailed Markdown↔LaTeX conversion rules. Loaded only during a format switch.

Command-only details currently in `commands/init.md`, `commands/format.md`, `commands/export.md`, etc. get folded into these references.

Then:
- **Commands become thin wrappers** (~5 lines): load the `quill-context` skill and execute the named workflow for `$ARGUMENTS`. Frontmatter (description, argument-hint, allowed-tools) stays so discoverability and tool scoping are preserved.
- **Agent keeps its persona section** (the "thoughtful collaborator" voice) but drops the restated mechanics, pointing at the skill instead.
- **Codex** already consumes `workflows.md`; it now reads the same canonical file Claude Code does, so divergence is structurally impossible.

## 2. State restructure (closes the compression leak)

### New project layout

```
project-root/
├── quill.json              ← stable metadata + bounded indexes (stays small)
├── .quill/
│   └── summaries/
│       ├── ch-01.json      ← per-chapter summary + ending_excerpt
│       ├── ch-02.json
│       └── ...
├── chapters/
├── scripts/
├── characters/  (fiction)  |  concepts/  (nonfiction/technical)
└── export/
```

`.quill/` is **committed** with the book project (must not be gitignored). The generated project README notes this.

### quill.json changes

- **Removed:** `chapter_summaries`, `last_chapter_ending`.
- **Added:** `schema_version: 2`.
- Everything else stays (metadata, structure, world_rules, continuity_flags, characters/concepts, outline, open/resolved threads, last_chapter_written, revision_log).
- Result: `quill.json` stays small and each per-chapter rewrite is tiny → far less token cost and a much smaller corruption blast radius.

### Per-chapter summary file `.quill/summaries/ch-NN.json`

```json
{
  "chapter": 5,
  "what_happened_or_covered": "2-3 sentence summary",
  "emotional_beat_or_takeaway": "key beat",
  "new_characters_or_concepts": [],
  "new_world_facts": [],
  "threads_or_questions_opened": [],
  "threads_or_questions_closed": [],
  "actual_word_count": 3800,
  "ending_excerpt": "final ~150 words verbatim"
}
```

### Briefing build (write/revise)

1. Read `quill.json` (small now).
2. Read only the summary files for the **3 nearest prior written chapters** by number.
3. For prose handoff, read the **actual predecessor's** `ending_excerpt` (chapter `N-1` if written, else the nearest lower written chapter).
4. Never load full chapter prose unless the user explicitly asks.

This **deletes the entire `last_chapter_ending` special-casing**. The previous "it refers to the highest-numbered chapter, not necessarily the predecessor" caveat (repeated in SKILL.md, workflows.md, write.md, revise.md) disappears — non-linear writing now always pulls the correct ending.

### Compaction

- Still applies to the bounded arrays in `quill.json` (resolved_threads ≤10, revision_log ≤10, continuity_flags ≤20, world_rules dedup).
- The 3-sentence cap on `what_happened_or_covered` stays (keeps briefings lean).
- Summaries no longer need the "keep quill.json manageable" justification — they are separate files.

### Migration (v1 → v2)

`quill-validate.py --migrate`:
- Detects a v1 file (no `schema_version`).
- Moves each `chapter_summaries[N]` to `.quill/summaries/ch-NN.json`.
- Seeds the last written chapter's `ending_excerpt` from the old `last_chapter_ending`; other chapters get `ending_excerpt: ""` (optionally backfilled from chapter files if present).
- Sets `schema_version: 2`, removes `chapter_summaries` and `last_chapter_ending`.
- Idempotent: re-running on a v2 project is a no-op.

## 3. Tooling + correctness

Scripts are bundled in `skills/quill-context/templates/` and copied into the project's `scripts/` at init (same pattern as today). **All scripts are optional** — every workflow keeps a "if shell execution is unavailable, do the equivalent directly" fallback.

- **`quill-sync-outline.py`** — unchanged behavior.
- **`quill-validate.py`** (new) — validates `quill.json` and summary files against the schema; checks referential integrity:
  - every `written`/`revised` outline entry has a chapter file **and** a summary file;
  - every summary file maps to a real outline entry;
  - thread IDs are unique; open/resolved sets are disjoint;
  - `last_chapter_written` is consistent with summaries present.
  - `--migrate` performs the v1→v2 transform. Exits non-zero on hard errors.
- **`quill-stats.py`** (new) — computes **accurate** word counts from chapter files (strips LaTeX/Markdown markup reasonably), plus progress % and pace estimate. The write workflow uses it to set `actual_word_count` (one true number); `/status` uses it for progress. LLM estimate is the fallback when no shell.
- **`tests/`** (pytest) — covers the pure functions: `is_stub`, `latex_escape`, `render_stub`, the migration transform, word counting, and validation rules. This is the bug class the recent manual `latex_escape` fix addressed.
- **GitHub Actions CI** — a small workflow runs `pytest` on push/PR.

---

## 4. Polish + consistency

- **"Quill" everywhere** — update `agents/openai.yaml` `display_name` from "Quill Book Studio" to "Quill"; verify plugin.json/README/commands (already "Quill").
- **`allowed-tools` audit:**
  - `SKILL.md` → `Read, Write, Bash` (workflows run python/pandoc/pdflatex).
  - Per-command: write/revise/status gain `Bash` (for `quill-stats.py`); init/outline/export/format already have it; threads/character stay `Read, Write` unless they need sync.
- **Activation wording** — reword the skill `description` to be accurate for both hosts: trigger on intent (initialize/write/revise/outline/threads/character/format/export a book with Quill) with the presence of `quill.json` as a hint, since Claude Code skills activate by description match rather than file detection.

---

## Edge cases

- Missing summary file for a written chapter → `validate` flags it; write/revise degrade gracefully (rebuild from the chapter file if present).
- `.quill/` missing → recreated on next write.
- Partial write → blast radius is one small summary file + a tiny `quill.json` delta.
- Mixed `.md`/`.tex` for one chapter → existing format-switch guard still stops and asks.

---

## Testing strategy

- **Unit (pytest):** all pure script functions + migration transform + validation rules.
- **CI:** GitHub Actions runs pytest on push/PR.
- **Manual smoke test (both formats):** init → write 1 → write 3 (non-linear) → revise 1 → status → export; verify briefings pull the correct predecessor `ending_excerpt`, `quill.json` stays small, and summary files are created/updated correctly.
- **Migration test:** take a v1 `quill.json` fixture, run `--migrate`, assert v2 layout and idempotency.

---

## Out of scope (YAGNI)

- No external API calls; Claude/Codex remains the LLM.
- No new book types or output formats.
- No rewrite of the format-conversion logic itself (only relocating it to a reference file).
- No backward-compat shim beyond the one-time migration (project is pre-1.0).

---

## Build order (high level)

1. References split: create `schema.md`, `init-templates.md`, `format-conversion.md`; make `workflows.md` canonical and fold in command-only detail.
2. State restructure in the workflows + schema (per-chapter summaries, `ending_excerpt`, `schema_version`).
3. Thin out commands and the agent to point at the skill.
4. Add `quill-validate.py` (+ migration), `quill-stats.py`; wire workflows to use them with fallbacks.
5. Add `tests/` + GitHub Actions CI.
6. Polish pass: naming, `allowed-tools`, activation wording.
7. Update top-level README to reflect the new layout and `schema_version`.
